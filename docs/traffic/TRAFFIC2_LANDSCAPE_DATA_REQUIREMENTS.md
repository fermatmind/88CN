# TRAFFIC2 Landscape Data Requirements

Result: GO_TRAFFIC2B

## Scope

This document defines data requirements for a future `/landscape` implementation. It does not create data access code, modify schemas, run queries, change Supabase, or mutate the data repository.

## Safe Data Sources

Future `/landscape` implementation may read:

- reviewed/published project records;
- existing finite registries for stacks;
- existing finite registries for collections;
- existing finite registries for verticals;
- existing approved alternatives;
- reviewed reports registry;
- public signal checks already stored or generated by local safe checks;
- source-linked public metadata;
- route-safe aggregate counts.

## Forbidden Data Sources

Future `/landscape` must not read or render:

- submitted private data;
- pending review records;
- quarantined records;
- scouted private records;
- rejected records;
- admin notes;
- founder private contact data;
- payment state;
- API credential data;
- metering data;
- buyer-interest data;
- raw database rows;
- private telemetry;
- internal Signal Score internals;
- internal Source Confidence internals if not public-safe.

## Allowed Fields

| Field | Public display rule |
| --- | --- |
| project name | Allowed for reviewed/published projects. |
| slug | Allowed for safe project links. |
| category | Allowed when reviewed. |
| short public description | Allowed only if reviewed. |
| website URL | Allowed if public and source-backed. |
| GitHub URL | Allowed if public. |
| docs URL | Allowed if public. |
| launch page | Allowed if public. |
| stack tags | Allowed if finite and reviewed. |
| collection tags | Allowed if finite and reviewed. |
| vertical tags | Allowed if finite and reviewed. |
| lifecycle label | Allowed for published/claimed/archived when safe. |
| last reviewed date | Allowed when source-backed. |
| website reachable | Allowed as detection evidence. |
| sitemap detected | Allowed as detection evidence. |
| JSON-LD detected | Allowed as detection evidence. |
| canonical detected | Allowed as detection evidence. |
| source-backed open-source/commercial status | Allowed only when source-backed. |
| reviewed sample count | Allowed; must not be framed as global coverage. |

## Forbidden Fields

Never render:

- private contact fields;
- admin notes;
- review notes;
- payment state;
- customer/payment-provider fields;
- API credential data;
- metering data;
- buyer-interest data;
- private analytics;
- internal credential material;
- raw payload;
- raw database row;
- submitted/private records;
- quarantined records;
- rejected records;
- private scouted records;
- unreviewed claims;
- fake global market counts.

## Lifecycle Policy

Allowed for active landscape display:

- `published`
- `claimed`
- `owner_verified`

Conditional:

- `archived`, only as a clearly marked historical/status item and not as active coverage.

Forbidden:

- `submitted`
- `pending_review`
- `approved` when not yet published
- `scouted`
- `quarantined`
- `rejected`
- draft-only
- private/admin-only
- unreviewed claims

## Aggregation Rules

- Count only reviewed/published public-safe records.
- Label counts as reviewed samples.
- Do not show global market estimates.
- Do not combine private or pending records into public totals.
- Do not allow paid/Featured Signals to change counts, sorting, sitemap eligibility, source confidence, or score.
